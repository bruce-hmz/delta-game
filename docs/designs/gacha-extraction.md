## Vision

### 10x Check
A gacha platform that mirrors the CS:GO case ecosystem on mobile web. Multiple crate types, collection gallery, social sharing pipeline, pull statistics, and streak mechanics. Each 8-second pull produces a shareable vertical video clip for Douyin. The 10x version supports speedrun opening (10+ rapid pulls), clip compilation, and community comparison.

### Platonic Ideal
User opens page. Sees a military crate. Taps it. 3-second animation with frame-perfect CS:GO case opening rhythm: click, click, click... BANG. Loot card flips. Quality revealed. Rare drops trigger haptic vibration, particle explosion, auto-recorded clip. Share prompt appears with Douyin-ready caption. 8 seconds total. Tap again. No loading, no zone selection, no text walls. Pure dopamine loop.

## Engineering Review Decisions (2026-04-15)
- Migration: Phase approach (git init → Vercel migration → gacha rewrite)
- No real users — clean DB wipe acceptable
- Pull RPC: Use SELECT FOR UPDATE or atomic UPDATE...RETURNING for pity counter
- Guest auth: Server-generated UUID stored in httpOnly cookie + HMAC signature. Server validates cookie signature on every request. No client-side access to token (prevents XSS session hijacking).
- Guest sync: Server-authoritative. Client sends only idempotency keys + crate IDs. Server re-generates drops using stored random seeds (idempotency key as seed). Never trust client-provided item data. Recalculate pity server-side from authoritative pull history.
- API contract: Finalize all 7 endpoint request/response shapes BEFORE frontend work starts

### API Contract (7 endpoints)

#### POST /api/auth/guest
```
Request: {}
Response 200: { sessionId: string, ticketsRemaining: number, dailyLimit: number }
Response 409: { error: "session_exists", sessionId: string }
```

#### GET /api/gacha/crates
```
Request: {}
Response 200: {
  crates: Array<{
    id: string,
    name: string,
    description: string,
    imageUrl: string,
    dropRates: { white: number, blue: number, purple: number, red: number, gold: number },
    ticketCost: number,
    starRating: number
  }>
}
```

#### POST /api/gacha/pull
```
Request: { crateId: string, idempotencyKey: string }
Response 200: {
  item: {
    id: string,
    name: string,
    quality: "white"|"blue"|"purple"|"red"|"gold",
    imageUrl: string,
    value: number,
    affixes: Array<{ type: string, description: string }>
  },
  ticketsRemaining: number,
  pityProgress: { current: number, target: number },
  isPityTriggered: boolean
}
Response 402: { error: "no_tickets", ticketsRemaining: 0, dailyLimit: number }
Response 404: { error: "crate_not_found" }
Response 409: { error: "duplicate_request", item: {...} } // same idempotency key
```

#### GET /api/gacha/collection
```
Request: ?page=1&limit=20&quality=red&sort=recent
Response 200: {
  items: Array<{
    id: string,
    itemId: string,
    name: string,
    quality: string,
    imageUrl: string,
    value: number,
    pulledAt: string // ISO 8601
  }>,
  total: number,
  page: number,
  hasMore: boolean
}
```

#### GET /api/gacha/stats
```
Request: {}
Response 200: {
  totalPulls: number,
  pullsToday: number,
  dailyLimit: number,
  streak: { current: number, longest: number, lastPullDate: string },
  pityProgress: { current: number, target: number },
  qualityBreakdown: { white: number, blue: number, purple: number, red: number, gold: number },
  recentPulls: Array<{ name: string, quality: string, pulledAt: string }> // last 10
}
```

#### POST /api/gacha/sync
```
Request: { pulls: Array<{ idempotencyKey: string, crateId: string }> }
Response 200: {
  synced: number,
  skipped: number, // duplicates
  pityProgress: { current: number, target: number } // recalculated server-side
}
Note: Server re-generates drop results using idempotencyKey as random seed. Client never sends item data.
```

#### POST /api/auth/upgrade
```
Request: { phone?: string, wechatId?: string }
Response 200: { playerId: string, ticketsRemaining: number, dailyLimit: number }
Response 400: { error: "invalid_input" }
```

### Database Schema (new gacha tables)

```
┌─────────────┐     ┌──────────────┐     ┌────────────────┐
│crate_configs │     │ pull_history  │     │ guest_sessions │
├─────────────┤     ├──────────────┤     ├────────────────┤
│id uuid PK   │◄──┐ │id uuid PK    │     │id uuid PK      │
│name text    │   │ │player_id uuid│◄──┐ │created_at ts   │
│description  │   │ │crate_id uuid │───┘ │last_active ts  │
│image_url    │   │ │item_id uuid  │     │ip_hash text    │
│star_rating  │   │ │quality enum  │     │user_agent text │
│ticket_cost  │   │ │idempotency_key│    │pulls_today int │
│drop_rates   │   │ │created_at ts │     │daily_limit int │
│  white float│   │ │value int     │     └────────────────┘
│  blue float │   │ │affixes jsonb │
│  purple flt │   │ │INDEX:        │     ┌────────────────┐
│  red float  │   │ │ player+crate │     │ player_streaks │
│  gold float │   │ │ created_at   │     ├────────────────┤
│active bool  │   │ │ idempotency  │     │player_id uuid  │
│sort_order   │   └─│ UNIQUE       │     │current_streak  │
│created_at   │     └──────────────┘     │longest_streak  │
└─────────────┘                           │last_pull_date  │
                                          │fail_streak int │
                                          │pity_count int  │
                                          │total_pulls int │
                                          │INDEX:          │
                                          │ player_id PK   │
                                          └────────────────┘
```

Key constraints:
- `pull_history.idempotency_key` UNIQUE per player (prevents double-pull)
- `pull_history` indexed on `(player_id, created_at DESC)` for collection pagination
- `pull_history.quality` is ENUM: white, blue, purple, red, gold
- `crate_configs.drop_rates` CHECK: all values >= 0 AND sum = 1.0
- `player_streaks.pity_count` incremented atomically via UPDATE...RETURNING
- `guest_sessions` cleaned up after 7 days of inactivity
- All monetization enforcement server-side: ticket count checked BEFORE pull, decremented atomically in the same transaction as the pull insert
- Coze removal: Dedicated checklist (4 SDK imports, 7 env var files, server.ts, build scripts, analytics)
- Drizzle ORM: Use as the sole migration system (drizzle-kit push)
- Tests: Vitest. Critical tests written alongside implementation.
- iOS Safari: No haptics. Feature-detect navigator.vibrate.
- MediaRecorder: Use pause/resume. PNG fallback on Safari.
- Pity counter SQL (single atomic operation):
  ```sql
  WITH updated AS (
    UPDATE player_streaks
    SET pity_count = CASE WHEN pity_count >= 49 THEN 0 ELSE pity_count + 1 END
    WHERE player_id = $1
    RETURNING pity_count, (pity_count >= 49) AS pity_triggered
  )
  SELECT * FROM updated;
  -- If pity_triggered = true: override random drop with red quality
  -- If pity_triggered = false AND random roll = red: still reset pity_count to 0
  ```
- Safari share: PNG share card is server-generated from authoritative pull data, not a client screenshot. Uses canvas API to compose logo + item art + quality colors + QR code. Works on all browsers.
- First pull audio: pre-load Howler.js sound sprites on page load (before user interaction). If audio not loaded when first pull happens, play silently (no delayed audio).

### Accessibility Specification
- **Color contrast:** All text on dark bg must meet WCAG AA (4.5:1 for body, 3:1 for large text). White (#FFF) on #0a0a0a = 19.3:1. Secondary text (#888) on #0a0a0a = 5.2:1. PASS.
- **Quality color contrast:** Quality colors on dark bg: white 19.3:1, blue 3.8:1 (borderline — use blue text only for labels at 16px+, not body), purple 3.2:1 (use only for decorative borders and badges, never for text), red 4.1:1, gold 8.4:1.
- **Keyboard nav:** Tab order: crate swipe → OPEN button → Collection tab → Stats tab → Pity counter. All CTAs focusable. Enter/Space triggers buttons. Escape closes overlays/modals.
- **Screen reader:** aria-live="polite" on loot reveal (announces quality + name). aria-label on crate cards (name + rarity tier). aria-valuenow on pity progress. Collection items: name + quality as alt text.
- **Focus visible:** 2px solid ring in quality color on focused element. Default browser outline removed, custom focus ring added.
- **Touch targets:** All interactive elements minimum 44x44px. OPEN button is 56px. Collection cards are 160x200px. Nav tabs are 48px height.
- **Motion:** `prefers-reduced-motion: reduce` → skip all shake/spin/particle animations. Show static card flip (CSS transform only, 200ms). No haptics in reduced-motion mode.
- **Announcements:** New pull result announced via aria-live. Error states announced via aria-live="assertive". Loading states: aria-busy="true" on container.
- **Safari-specific:** No haptics (feature detect). No MediaRecorder in some Safari versions → PNG fallback. Respect Safari safe area insets via env(safe-area-inset-*).

- Recovery: no auto-retry animation. User sees the static result and can pull again.

| 4 | Hits 10 whites | "Is this rigged?" | Probability disclosure accessible from top bar (always visible), showing exact rates. Transparency builds trust |
| 5 | Guest hits 3-pull limit | Want more, blocked | "注册获取5次免费" CTA is warm, not paywalled. Show what they COULD get: preview a red item card at 20% opacity |
| 6 | Share fails on red drop | Angry, trust broken | PNG fallback auto-generates. Toast: "已保存截图, 可手动分享". Never lose the moment |
| 7 | Returns next day after bad streak | Skeptical | Streak fire persists, pity counter preserved (shows progress). "昨天开了10箱, 今天运气会更好!" |

### Emotional Guardrails
- Never show "0 items" in any counter. Empty states always have a CTA.
- Pity counter: only visible after 1+ pull (don't remind new users about failure)
- After 5+ whites: subtle glow animation on "OPEN" button (temptation cue)
- After red/gold: 2-second "celebration freeze" before showing CTAs (let the moment breathe)
- Share prompt timing: appear 500ms after reveal completes, not during animation
