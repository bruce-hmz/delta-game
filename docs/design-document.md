Created: 2026-04-17

## 1. 产品定位

**一句话：** 手机网页上的策略搜刮游戏，还原三角洲"搜刮-战斗-撤离"的核心紧张感。

**目标用户：** 三角洲系列玩家，因时间/地点/设备限制无法玩原版，但有 5-30 分钟碎片时间。

**核心体验：** 搜刮紧张感 + 回合制战斗 + 策略规划 + 收集养成

**平台：** 移动端网页优先（Next.js + Canvas），兼容桌面浏览器

---

## 2. 核心循环

```
选择行动区域 → 部署（选装备）→ 探索地图节点 → 搜刮/战斗/事件 → 撤离决策 → 结算
                                      ↑_______________↓
                                    （继续深入 or 撤离）
```

每一局是一次"行动"。玩家在地图上逐个探索节点，搜刮物品，遭遇敌人。
核心决策：**什么时候撤离？继续探索奖励更好，但死亡风险增加。**
死亡 = 失去本次所有搜刮物品。

## 3. 地图系统

### 3.1 节点类型

| 类型 | 图标颜色 | 描述 |
|------|---------|------|
| 搜刮点 | 绿色 | 获得随机物品，有概率触发陷阱（扣血） |
| 战斗点 | 红色 | 遭遇敌人，必须回合制战斗 |
| 事件点 | 黄色 | 随机事件（商人、受伤NPC、空投补给） |
| 撤离点 | 蓝色 | 可以选择撤离，带着物品安全离开 |
| 未知点 | 灰色 | 接近后才揭示类型，增加不确定感 |

### 3.2 地图生成

每局随机生成。节点数量和敌人密度由区域决定。
相邻节点之间有连接线，玩家只能移动到相邻节点。
起始点固定，撤离点位置随机（通常在地图边缘）。

### 3.3 行动区域

| 区域名 | 风险 | 节点数 | 战斗数 | 品质偏向 | 适合模式 |
|--------|------|--------|--------|---------|---------|
| 哈夫克发电站 | 低 | 5-8 | 1-2 | 白/蓝为主 | 快速行动 |
| 零号大坝 | 中 | 8-12 | 3-5 | 蓝/紫为主 | 快速/深度 |
| 黑鹰坠落区 | 高 | 12-16 | 5-7 | 紫/红/金概率提升 | 深度渗透 |
| 极光实验室 | 极高 | 16-20 | 7-10 | 红/金概率高，有独特掉落 | 深度渗透 |

## 4. 回合制战斗

### 4.1 玩家行动

每回合选择一个动作：

| 动作 | 效果 |
|------|------|
| 攻击 | 造成武器基础伤害 + 随机浮动（±20%） |
| 防御 | 减少下次受到伤害 50%，持续到下回合 |
| 使用道具 | 消耗背包物品（医疗包回血、手雷造成伤害等） |
| 逃跑 | 60% 概率成功脱离战斗；失败则承受一次攻击 |

### 4.2 敌人类型

| 敌人 | 血量 | 攻击力 | 行为模式 |
|------|------|--------|---------|
| 突击兵 | 30-50 | 8-12 | 每回合固定攻击 |
| 狙击手 | 20-35 | 15-25 | 蓄力1回合，下一回合造成双倍伤害 |
| 重装兵 | 60-100 | 5-8 | 血厚攻击低，需要多回合击杀 |
| 精英兵 | 50-70 | 12-18 | 随机切换攻击/防御 |
| Boss | 100-200 | 15-30 | 多阶段行为，有特殊技能 |

### 4.3 战斗结算

- 胜利：获得经验值 + 搜刮该区域物品
- 失败（血量归零）：失去本次行动所有搜刮物品，强制结束行动
- 逃跑成功：保留当前物品，回到地图继续探索

## 5. 搜刮与物品系统

### 5.1 品质体系（复用现有）

| 品质 | 颜色 | 搜刮概率（低风险区） | 物品类型 |
|------|------|---------------------|---------|
| 普通 | 白 | 55% | 基础弹药、医疗包、简易护甲 |
| 精良 | 蓝 | 28% | 瞄准镜、消音器、战术背心 |
| 稀有 | 紫 | 12% | 高级护甲、医疗站、EMP手雷 |
| 史诗 | 红 | 4% | 传奇武器部件、纳米修复、能量护盾 |
| 传说 | 金 | 1% | 终极武器蓝图、远古科技、神话装备 |

### 5.2 物品属性

每件物品包含：
- **基础价值**：对应品质的价值区间
- **词缀**（affix）：随机附加属性（价值+N%、幸运+N、开红概率+N%）
- **类型标签**：武器/防具/道具/材料，影响战斗中的用途

### 5.3 背包与负重

- 背包容量：8-12 格（根据装备扩展）
- 每件物品占 1 格
- 背包满后只能丢弃旧物品换新物品（策略决策）
- 负重不影响移动，但影响战斗中的闪避率

## 6. 撤离与死亡

### 6.1 撤离机制

- 地图上有 1-3 个撤离点
- 到达撤离点后可以选择撤离
- 撤离需要 1 回合"等待直升机"（期间可能遭遇追击）
- 成功撤离 = 所有搜刮物品入库

### 6.2 死亡惩罚

- 死亡 = 失去本次行动的所有搜刮物品
- 保留：经验值、任务进度、保底计数
- 连续死亡 3 次：触发"救援补给"，下次行动赠送基础装备

### 6.3 核心紧张感来源

> **"我再搜一个节点...还是现在撤？"**

这个决策贯穿每一局。搜刮越多物品越好，但死亡风险同步增加。
撤离太早 = 浪费机会；撤离太晚 = 可能全丢。这是游戏的核心多巴胺来源。

## 7. 双模式设计

### 7.1 快速行动（3-5 分钟）

- 1 个小区域，5-8 个节点
- 1-2 场战斗，简单 AI
- 搜刮物品品质偏基础
- 适合地铁/排队/碎片时间

### 7.2 深度渗透（15-30 分钟）

- 3 个区域串联，20+ 节点
- 难度递增，最后有 Boss 战
- 独特高价值搜刮物（仅深度模式掉落）
- 需要策略性管理血量、道具、背包空间
- 适合坐下来认真玩

## 8. 角色成长系统

### 8.1 经验与等级

- 每次行动获得经验（基于搜刮物品数量、击杀数、是否成功撤离）
- 升级解锁：新区域、背包扩容、基础属性提升

### 8.2 装备库

- 搜刮到的物品存入装备库
- 每次行动前选择装备带入（武器、防具、道具）
- 带入的装备在行动中可使用/消耗

### 8.3 每日系统

| | 游客 | 注册用户 | 月卡用户 |
|--|------|---------|---------|
| 快速行动 | 3次/天 | 5次/天 | 无限 |
| 深度渗透 | 不开放 | 2次/天 | 5次/天 |
| 背包容量 | 8格 | 10格 | 12格 |
| 保底计数 | 50次 | 50次 | 30次 |

## 9. 商业化路径

### 9.1 第一阶段（上线期）

- 免费游玩，游客 3 次/天
- 注册解锁更多次数和深度模式
- **零付费墙**，所有内容免费体验

### 9.2 第二阶段（商业化）

- **行动通行证**（月卡）：额外行动次数 + 独特区域 + 保底优化
- **军火商补给箱**：保留 gacha 机制，开出装备/外观/蓝图
- **外观商城**：武器皮肤、角色外观（纯装饰，不影响数值）
- **季度更新**：新区域、新敌人、新物品池

### 9.3 第三阶段（社交化）

- 排行榜：单局搜刮价值排行、无死亡通关速度排行
- 装备展示：玩家间分享收藏
- 公会系统：组队挑战高难度区域

## 10. 技术架构

### 10.1 前端

- **框架**：Next.js（现有）
- **地图渲染**：Canvas 2D（节点式地图，不需要复杂物理）
- **战斗动画**：CSS 动画 + 简单 Canvas 特效
- **移动优先**：触控操作，44px 最小触控区域

### 10.2 后端

- **数据库**：Supabase + Drizzle ORM（现有）
- **新增表**：`runs`（行动记录，含地图JSON）、`run_inventory`（行动中背包）、`equipment_stash`（持久装备库）、`enemy_templates`（敌人配置）
- **API**：Next.js API Routes（现有模式）
- **状态权威**：服务器是唯一真值源，IndexedDB 仅用于断线恢复
- **地图存储**：地图数据作为 JSON 嵌入 runs 表，单次查询获取完整状态
- **缓存**：Crate 配置和敌人模板使用 TTL 缓存（1-5分钟），避免每次请求查询数据库

### 10.3 复用现有代码

| 现有模块 | 复用方式 |
|---------|---------|
| 品质系统（quality） | 直接复用，重命名物品池 |
| 物品生成（gacha-service） | 改造为搜刮物品生成器 |
| 保底机制（pity） | 复用到搜刮系统中 |
| 用户系统（auth） | 直接复用 |
| 每日限制（daily limit） | 复用，从开箱次数改为行动次数 |
| 数据库 schema | 扩展，新增地图/战斗相关表 |

## 14. 工程评审决定

以下决定由 `/plan-eng-review` (2026-04-17) 锁定：

### 架构决定

1. **服务器生成地图**：开局时服务器生成地图，存储为 JSON 在 runs 表中。客户端通过渐进式披露接收节点类型（仅访问过的 + 相邻节点）。
2. **服务器权威战斗**：客户端提交动作选择，服务器解析完整回合。客户端从不计算伤害。
3. **IndexedDB 仅用于崩溃恢复**：客户端为重新加载后恢复保存状态，但永远不会领先于服务器。死亡首先写入服务器。
4. **基于回合的撤离**：无实时计时器。撤离是基于回合的"等待直升机"动作，可能遭遇敌人攻击。
5. **并发行动阻止**：数据库约束 `UNIQUE (player_id) WHERE status = 'active'`。
6. **地图嵌入在 runs 行中**：地图数据作为 JSON 列存储，用于单次查询访问。

### 性能要求

- 同步路由 N+1 查询必须在上线前修复（批量幂等键检查）
- Crate 配置必须缓存（TTL 1-5 分钟）
- 连接池大小至少 20 个连接用于生产环境
- 每次拉动时将 `pity_count` 增量存储在 `player_streaks` 中

### 战斗系统规范

| 敌人 | 行为决策树 |
|------|-----------|
| 突击兵 | 每回合固定攻击 |
| 狙击手 | 蓄力1回合（标记），下一回合双倍伤害 |
| 重装兵 | 每回合攻击（低伤害），需要多回合击杀 |
| 精英兵 | 随机切换攻击/防御（50/50概率） |
| Boss | 多阶段：Phase 1 (攻击为主) → Phase 2 (混合攻击/特殊技能) → Phase 3 (狂暴，全攻击) |

### 词缀系统扩展

物品词缀需要包含战斗效果：
- **经济类**（现有）：价值+N%、幸运+N%、开红概率+N%
- **战斗类**（新增）：伤害+N%、闪避+N%、暴击+N%、防御+N%
- 词缀池按物品类型分配：武器偏战斗词缀，防具偏防御词缀，道具偏经济词缀

## 13. 成功标准

- 留存率：次日留存 > 30%，7日留存 > 10%
- 单局时长：快速行动平均 4 分钟，深度渗透平均 18 分钟
- 付费转化率 > 3%（月卡用户）
- 日活跃用户在三角洲相关社群传播后 7 天内突破 1000
